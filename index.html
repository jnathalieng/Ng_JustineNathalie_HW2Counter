OVERVIEW

This counter is a reusable JavaScript class that controls a number inside a container. It lets the user increment, decrement, and reset the value, but everything is handled inside one class instead of writing the same code over and over again.

The idea here is to avoid repeating logic every time I need a counter on a page. Instead of manually adding event listeners and updating the UI each time, I can just instantiate the class and it works right away.

The counter uses object-oriented programming, where the state controls what is shown on the UI. For example, when the count reaches 0, the decrement and reset buttons become inactive so the user can't go below 0 or reset unnecessarily.

I also created a StepCounter version which extends the original class. This one works the same way but instead of increasing or decreasing by 1, it uses a custom step value. This shows how inheritance works and how I can reuse and modify existing logic without rewriting the whole thing.

Overall, this project is both a functional component and a documentation-style page showing how it works and how it can be reused.

HOW IT WORKS

The counter works by storing a value inside the class and updating the UI based on that value.

When the class is created, it selects the container using the selector and gets the elements inside it, like the display and buttons. It also sets up the event listeners for increment, decrement, and reset.

Each button triggers a method:

  • increment() : adds to the count
  • decrement() : subtracts but never goes below 0
  • reset() : brings back to the initial value and logs in the console

Instead of updating everything randomly, the UI changes are handled in one place using the update() method. This makes sure everything stays consistent.

The update method:

  • updates the displayed number
  • adds or removes the inactive class on the buttons
  • keeps the UI in sync with the current count

For the StepCounter, it works the same way but it overrides the increment and decrement methods. Instead of changing by 1, it uses a custom step value passed into the class.

So basically:

state (count) → update() → UI changes

Everything depends on the value of the count.

LIVE DEMO

Below are multiple instances of the counter running on the same page. Each one is created using the same class but with different initial values (0, 10, 100, and 1000) to show how flexible and reusable the component is.

The first two counters (0 and 10) use the base Counter class and behave like standard counters. They increment and decrement by 1, and cannot go below 0. The decrement and reset buttons also become inactive when needed.

The next two counters (100 and 1000) use the StepCounter class. Instead of increasing or decreasing by 1, they use a custom step value, which allows the counter to change in larger increments. This demonstrates how the original class can be extended and modified without rewriting the entire logic.

All counters work independently, even though they share the same underlying logic.

HOW TO USE IT

To use the counter, I first need a container in the HTML for each instance. After that, I instantiate the class in JavaScript and pass in the selector plus the initial value I want the counter to start with. For the StepCounter, I also pass in a third parameter for the step value.

Example HTML container markup

<div id="counter-container1" class="counter col-span-full m-col-span-6"></div>

Example instantiation code

import { Counter, StepCounter } from './counter.js';
                        
                        const counter1 = new Counter('#counter-container1', 0);
                        const counter2 = new Counter('#counter-container2', 10);
                        const counter3 = new StepCounter('#counter-container3', 100, 100);
                        const counter4 = new StepCounter('#counter-container4', 1000, 1000);

Parameters

  • selector: targets the counter container in the HTML
  • initialValue :sets the starting value of the counter
  • step : only used in StepCounter, and controls how much the value increases or decreases each time

This makes the component reusable because I can create multiple counters with different starting values and different behaviors without rewriting the whole logic each time.
